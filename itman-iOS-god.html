iOS - 最成熟的移动商业平台

iOS多年来已经成熟了很多。在第9次升级中，Apple在iPad上引入了一种名为San Francisco，3D Touch和多任务的新系统字体。在Xcode中，你会发现Stack Views，这是一个很棒的工具，可以让你的布局更具适应性而不需要自动布局的复杂性。 Apple比以往任何时候都更加鼓励自适应布局，以便您的设计可以在多个设备上运行。

自适应布局和多任务处理

随着要处理的设备分辨率越来越高，使布局自适应至关重要。 使用Sketch Constraints或Xcode等工具，您必须以一种灵活的屏幕尺寸设计，并在需要时显示额外的菜单.

以下是布局如何适应从iPhone到iPad Pro。 用户界面扩展而不是扩展。 对于横向模式下较大的屏幕（如iPad和iPhone 8 Plus），将显示左侧导航而不是标签栏。

SF字体

默认字体现在是旧金山字体，由Apple内部制作。 我鼓励您观看视频 ，了解它如何影响iOS的设计。

SF字体跟踪

iOS会根据字体大小自动调整跟踪值和旧金山的文本/显示。 这确保了字体总是易于阅读。 在20磅或更高时，应使用SF UI显示 ，否则使用SF UI文本 。 这些跟踪值只能在Photoshop中应用，但这是在Sketch中转换的公式。

使用此Sketch插件快速应用正确的字符间距值。

3D 触控

iOS中的一项重要新功能称为3D Touch，允许用户快速访问应用内外的选项。

用户现在可以用力按下您的应用程序图标并查找常用项目。 在应用程序内部，可以预览邮件，并且可以在进入全屏之前预览链接。

将3D Touch视为Mac上的键盘快捷键 - 它们使人们能够更快地完成重复任务。 您必须设计快捷方式，以提高高级用户的工作效率。 但就像键盘快捷键一样，基本功能不应该是3D Touch独有的。 没有它，您的用户必须能够正常操作您的应用程序。

点和像素

开发人员使用点值，因此了解像素的差异非常重要。 当iPhone首次推出时，两个单元是相同的：1pt等于1px。 然后当视网膜屏幕出现时，1pt变为2px。 因此，将点视为原始iPhone中的值，将像素视为实际值，具体取决于像素密度（iPhone 4,5,6 = @ 2x，iPhone 8 Plus = @ 3x）。

有关详细信息，请查看本指南。

iPhone 分辨率

iPhone有4种主要分辨率: 320 x 480 pt (iPhone 4), 320 x 568 pt (iPhone 5), 375 x 667 pt (iPhone 7) and 414 x 736 pt (iPhone 7 Plus). 布局不会缩放，但会根据分辨率进行扩展。 例如，导航栏仅调整宽度但保持相同的高度。 里面的元素保持完整。

iPhone 7 Plus是唯一一款在风景模式下更像iPad的iPhone。 换句话说，将出现左导航，取代Tab键的需要。

iPad 分辨率

iPad有2种主要分辨率: 768 x 1024 pt (iPad), 1024 x 1366 pt (iPad Pro).

iPad有2个新功能：Slide Over和Split View。 Slide Over 是一个叠加层，显示在屏幕右侧，不会影响当前应用的布局。

拆分视图允许用户通过在纵向模式下同时运行2个应用程序来执行多任务。

App图标

应用程序图标用于应用程序的品牌塑造。 这是用户体验它时首先看到的东西。 它在Spotlight和Settings中的主屏幕App Store中显得突出显示。

iPhone不再支持@ 1x格式，因此您无需生成它们。 应用程序图标现在有2个分辨率：@ 2x和@ 3x。 有3种类型：应用程序图标，聚光灯和设置。 对于iPad，使用@ 1x和@ 2x。

超椭圆

从iOS 7开始，圆角已从平圆角转变为超椭圆形。 请务必注意不要使用蒙版导出图标，否则您可能会发现黑色文物。 相反，只需将方形资产导出到App Store即可。

图标网格

Apple在他们的一些图标上应用了黄金比例。 这确保了图标是完美的，同时保持良好的比例。 虽然这是一个很好的规则，但这不是一个严格的规则。 甚至苹果也在许多图标上省略了它。

颜色

iOS使用鲜艳的颜色来显示按钮。 这些颜色倾向于在白色背景下与黑色背景一样好。 请记住，颜色应该稀疏地使用，用于号召性用语和最小的品牌区域，例如导航栏。 粗略地说，只有10-20％的设计应该有颜色，否则它们会与内容竞争太多。

iOS通常使用中性色作为背景和菜单区域。 使用与白色背景形成鲜明对比的黑色文本来使文本阅读舒适。 最后，柔和的蓝色用于使按钮突出。

按钮和字体大小

一般规则是按钮为44pt，12pt 小文本, 17pt 正文， 20pt+ 标题。

间距和对齐

一般规则是最小填充8pt。 这创造了足够的呼吸空间，这使得布局更容易扫描，文本更具可读性。 此外，UI元素应对齐，文本应具有相同的基线位置。

状态栏 Status Bar

建议在尽可能多的地方包含状态栏。 用户依赖它来获取信号，时间和电池等重要信息。 文本和图标可以是白色或黑色，但背景可以自定义为任何颜色，并与导航栏合并。

导航栏 Navigation Bar

导航栏用于快速了解屏幕信息。 左侧部分可用于放置“ 后退” ，“ 配置文件” ，“ 菜单”按钮，而右侧部分通常用于“ 添加” ，“ 编辑” ， “完成”等操作按钮。 请注意，如果您使用任何这些系统图标，则无需为它们创建资产。

就像状态栏一样，背景可以自定义为任何颜色，并且通常具有微妙的模糊，以确保文本始终可读。 状态栏存在时，两个背景都合并。

搜索栏 Search Bar

如果您有很多内容，那么将其设为可搜索是明智之举。

工具栏 Tool Bar

当您需要更多空间来放置操作按钮和屏幕状态时，您将需要使用工具栏。

标签栏 Tab Bar

标签栏是多个屏幕之间的主要导航。如果你要展示的东西很少请避开折叠菜单。 始终可见的菜单会增加使用量，因为显而易见的总是更胜一筹。 此外，我们鼓励在图标旁边添加文字，因为大多数人不会立即识别符号，特别是当它们不是普遍知道的时候。

当它们不被选中时，图标通常会有轮廓而不是填充。 像这样，他们吸引的注意力较少。

表视图 Table View

表视图是用于列出内容的非常常见的用户界面。 大多数应用程序使用表格视图的形式。 这是因为它可以是非常基本的，或者可以高度定制到最小的元素。

在基本级别，您可以使用许多预设样式和附件。

单元格也可以分组，上面有标题，下面有描述。

集合视图 Collection View

如果网格样式中包含行和列，则需要“集合视图”。 虽然更高级，但它几乎可以创造出您梦寐以求的任何布局。

集合视图布局可能看起来像这些，或它们的组合。 可能性是无止境。

呈现 Modals

提醒对话框用于传达关键信息并提示快速操作。 提醒应保持最小，退出必须明显。

活动对话框允许您共享iOS功能的内容（文本，图像，链接），如Airdrop，收藏夹，书签以及Mail，Facebook，Twitter等应用程序。 虽然外观无法定制，但选项是。

当显示的信息不短时，您可以设计一个全尺寸的模态，通常出现在幻灯片，淡入淡出，翻转或页面动画中。 与其他模态一样，它必须易于取消并尽可能短。

键盘

键盘用于在文本字段中输入信息，例如搜索，聊天或登录。 它是高度可定制的，适用于URL，电子邮件，电话号码甚至表情符号。 您可以在Light和Dark主题之间进行选择，以及如何命名操作按钮（ 返回是默认值)。

选择器 Picker

如果有多个选项可供选择，则可以使用Picker控件。 它对于日期来说特别有用，日期可以控制一个动作中的3个字段。

分段控制 Segmented Control

当标签栏导航到主要部分时，分段控件用于子部分。

滑块 Sliders

滑块是不太精确的交互式控件，但对于声音，亮度和视频进度等快速设置非常有用。

进度条 Progress

进度条是一个指示器，显示活动已经走了多远。 例如，您可以使用它来显示Web视图的加载进度。 请注意，高度可以自定义。

开关 Switch

使用此选项可在打开和关闭之间快速切换。 除开/关外，请勿将其用于其他任何用途。

步进 Stepper

Stepper比Slider更慢但更精确，允许用户以1的增量增加或减少一个值。 边框和背景可自定义。

iOS图标

这些是渗透iOS的原生图标。 由于它们是常用的，因此用户可以立即识别它们的含义。 将它们用于其他目的可能会使您的用户感到困惑，因此了解它们在iOS中的使用方式非常重要。

当您在这些之外设计自定义图标时，使用众所周知的符号很重要。 此外，我强烈建议他们随身携带10pt或更长的小文本。

资源

这些模板不仅可用于学习，还可用于重复使用和自定义，因此您无需从头开始。 随着您对它们的熟悉程度越来越高，您将能够获得创造力。

适用于iPhone的iOS 11 GUI

如果您正在为iOS设计，则需要使用预制元素，例如状态，导航和标签栏。 适应所有颜色，尺寸，字体和组件。

iPad GUI

新的iPad UI套件包括纵向和横向模式下的iPad Pro设备和键盘。 一切都在向量中。

Apple Watch GUI

Apple Watch非常完整的UI工具包，包括Apple Watch设备，矢量图标和时钟。 您可以以任何分辨率调整大小并导出它们。

UI设计的注意事项

有些做法应该不惜一切代价避免，特别是如果你是iOS新手。 请按照Apple提出的这些简单示例进行操作。
